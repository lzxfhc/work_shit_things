## 一、运行时组件调研

### 1. Computer Use / Browser Use / Mobile Use

#### 1.1 Computer Use (计算机使用)

| 维度 | 内容 |
|------|------|
| **代表产品** | Anthropic Claude Computer Use |
| **技术原理** | 通过视觉模型观察屏幕内容，模拟人类操作（移动光标、点击按钮、输入文字） |
| **核心能力** | 让AI像人一样使用计算机，执行各种标准工具和软件程序 |
| **应用场景** | 自动化重复流程、软件构建与测试、开放式研究任务 |
| **技术特点** | 不针对特定工具定制，而是教会AI通用的计算机操作技能 |
| **发布时间** | 2024年10月（公测），Claude 3.5 Sonnet首发 |
| **最新进展** | Claude Opus 4.5（2025年11月）被称为"世界上最好的编码、代理和计算机使用模型" |

## 二、高代码托管调研

### 1. 定义与概念

**高代码托管**：指为专业开发者提供的Agent开发能力，允许通过开放底层接口、支持自定义代码嵌入等方式，对复杂业务逻辑进行深度定制。与低代码/零代码形成"全民开发+专业攻坚"的互补生态。

### 2. 竞品对比表

| 平台 | 产品 | 高代码能力 | 主要特性 |
|------|------|------------|----------|
| **阿里云无影Bay** | AgentKit | ✓ SDK开发 | Go/Python/TypeScript SDK；MCP Server；自托管支持 |
| **字节Coze** | Coze Studio (开源) | ✓ 插件开发 | Apache 2.0协议；工作流引擎；插件框架；多语言SDK |
| **AWS** | Bedrock AgentCore | ✓ SDK开发 | 全托管Runtime；Browser/Code Interpreter工具；9个Region |
| **Azure** | Foundry Agent Service | ✓ Pro-code | CI/CD集成；多代理编排；OpenAPI支持；MCP协议 |
| **OpenAI** | AgentKit | ✓ SDK开发 | 可视化画布+代码开发；Codex内置；80%功能由Codex构建 |

## 三、建议与结论

### 1. 运行时组件建议

| 组件 | 建议方向 | 理由 |
|------|----------|------|
| **Computer Use** | 跟进Anthropic方案 | 唯一成熟的通用计算机使用能力，Claude 4.5已是业界标杆 |
| **Browser Use** | 自研+开源结合 | Browser-Use开源框架成熟（89%准确率），可基于此定制 |
| **Mobile Use** | 观望+试点 | 技术成熟度较低（最高43%成功率），建议2026年Q2再深入 |
| **Code Interpreter** | 优先建设 | AWS和Azure均已GA，是Agent基础能力的标配 |

### 2. 高代码托管建议

| 维度 | 建议 |
|------|------|
| **SDK语言** | 优先支持Python和TypeScript，覆盖主流开发者 |
| **协议支持** | 必须支持MCP协议，这是2025年Agent互联互通的标准 |
| **部署灵活性** | 同时支持云托管和自托管（参考Coze开源策略） |
| **企业特性** | VPC隔离、审计日志、RBAC权限必不可少 |

### 3. Q1 OKR建议项

| 优先级 | 项目 | 目标 |
|--------|------|------|
| P0 | Code Interpreter MVP | 支持Python/JS，15分钟执行时间，安全沙箱 |
| P0 | Browser Use集成 | 基于开源框架，支持Playwright，完成基础网页自动化 |
| P1 | 高代码SDK | Python SDK首版，支持MCP协议 |
| P2 | Computer Use预研 | 完成技术方案，Q2启动开发 |
| P2 | Mobile Use调研 | 持续跟踪，评估2026年路线图 |
